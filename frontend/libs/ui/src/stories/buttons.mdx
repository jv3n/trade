# Buttons

The design owned by `libs/ui/styles/components/_buttons.scss` is a **pure SCSS override** of Angular Material's MDC button. Touch the SCSS, every `<button mat-button>` / `mat-flat-button` / `mat-stroked-button` / `mat-icon-button` / `mat-raised-button` in the consumer app reflects the change — no HTML migration, no `UiButton` wrapper.

## How to use

1. Add `MatButtonModule` to the component's `imports` (it's a standalone import).
2. Use Material's regular attribute directives on a native `<button>` (or `<a>`) :

<Source
  language="ts"
  code={`import { Component } from '@angular/core';
import { MatButtonModule } from '@angular/material/button';

@Component({
  selector: 'app-my-feature',
  imports: [MatButtonModule],
  template: \`
    <button mat-flat-button (click)="save()">Save</button>
    <button mat-stroked-button (click)="cancel()">Cancel</button>
  \`,
})
export class MyFeature {}`}
/>

## Variant cheat-sheet

<Markdown>
{`
| Selector              | Material role           | Use it for                                                  |
| --------------------- | ----------------------- | ----------------------------------------------------------- |
| \`mat-flat-button\`     | Filled, no elevation    | **Primary CTA** — replaces the legacy \`.btn-primary\` class. |
| \`mat-stroked-button\`  | Outlined                | **Secondary CTA** — replaces the legacy \`.btn-secondary\`.  |
| \`mat-button\`          | Text only, low emphasis | Tertiary actions, inline toolbar buttons.                   |
| \`mat-icon-button\`     | Square icon, no label   | Toolbar / overflow / quick toggle.                          |
| \`mat-raised-button\`   | Filled + elevation      | Rare ; same palette as \`mat-flat-button\` here.             |
`}
</Markdown>

## Composing icon + label

Drop the icon and the label as direct siblings inside the button — Material projects the icon into the leading slot, label fills the default slot :

<Source
  language="html"
  code={`<button mat-flat-button>
  <mat-icon>auto_awesome</mat-icon>
  <span>Generate</span>
</button>`}
/>

**Caveat** (Material 21+) : if the icon is wrapped in a `@if` / `@else` block, the branch **must contain a single root node** matching the icon-slot selector. Otherwise the compiler raises `NG8011` and refuses to project. Split the conditional :

<Source
  language="html"
  code={`<button mat-flat-button [disabled]="loading()">
  @if (loading()) {
    <mat-spinner diameter="14"></mat-spinner>
  } @else {
    <mat-icon>refresh</mat-icon>
  }
  <span>{{ loading() ? 'Loading…' : 'Refresh' }}</span>
</button>`}
/>

## Tokens consumed

The SCSS rewrites Material's MDC tokens to point at the design-system custom properties — same palette dark + light, no per-variant logic to maintain :

<Markdown>
{`
| Variant      | MDC token override                                  | Resolves to            |
| ------------ | --------------------------------------------------- | ---------------------- |
| Flat         | \`--mdc-filled-button-container-color\`               | \`var(--color-accent)\`  |
| Flat         | \`--mdc-filled-button-label-text-color\`              | \`var(--color-on-accent)\`|
| Stroked      | \`--mdc-outlined-button-outline-color\`               | \`var(--color-border)\`  |
| Stroked      | \`--mdc-outlined-button-label-text-color\`            | \`var(--color-text)\`    |
| Text         | \`--mdc-text-button-label-text-color\`                | \`var(--color-text)\`    |
| Icon         | \`--mdc-icon-button-icon-color\`                      | \`var(--color-text-muted)\`|
| All          | \`--mdc-*-container-shape\`                           | \`var(--radius)\` (8 px) |
| All          | \`--mdc-*-container-height\`                          | 36 px                  |
`}
</Markdown>

Editing the design = editing two files :

- `libs/ui/styles/_tokens.scss` — the palette/radii (cascades to all consumers, dark + light)
- `libs/ui/styles/components/_buttons.scss` — the button-specific overrides
